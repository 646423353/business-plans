# 数据库部署文档

## 数据库架构

本项目使用以下数据库：

- **MySQL 8.0+** - 主数据库，存储用户、项目、消息等核心数据
- **Redis 7.0+** - 缓存数据库，用于会话管理、配额计数等

## MySQL 部署

### 方式一：Docker 部署（推荐）

#### 1. 拉取镜像

```bash
docker pull mysql:8.0
```

#### 2. 创建数据目录

```bash
mkdir -p /data/mysql
```

#### 3. 启动容器

```bash
docker run -d \
  --name mysql \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=your_root_password \
  -e MYSQL_DATABASE=ai_business_planner \
  -e MYSQL_CHARACTER_SET_SERVER=utf8mb4 \
  -e MYSQL_COLLATION_SERVER=utf8mb4_unicode_ci \
  -v /data/mysql:/var/lib/mysql \
  --restart=always \
  mysql:8.0
```

#### 4. 创建应用用户（可选但推荐）

```bash
docker exec -it mysql mysql -u root -p

# 在 MySQL 命令行中执行
CREATE USER 'business_planner'@'%' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON ai_business_planner.* TO 'business_planner'@'%';
FLUSH PRIVILEGES;
```

### 方式二：Linux 系统安装

#### Ubuntu/Debian

```bash
# 更新包索引
sudo apt update

# 安装 MySQL
sudo apt install mysql-server -y

# 启动服务
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全配置
sudo mysql_secure_installation
```

#### CentOS/RHEL

```bash
# 添加 MySQL 仓库
sudo yum install https://dev.mysql.com/get/mysql80-community-release-el7-3.noarch.rpm

# 安装 MySQL
sudo yum install mysql-server -y

# 启动服务
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 获取临时密码
sudo grep 'temporary password' /var/log/mysqld.log

# 安全配置
sudo mysql_secure_installation
```

### 方式三：云数据库服务

推荐使用云服务商提供的托管数据库：

- **阿里云**：RDS MySQL
- **腾讯云**：云数据库 MySQL
- **AWS**：Amazon RDS for MySQL
- **Azure**：Azure Database for MySQL

#### 配置要点

1. 选择 MySQL 8.0 版本
2. 设置字符集为 `utf8mb4`
3. 配置白名单允许后端服务器访问
4. 开启自动备份

## MySQL 配置优化

### my.cnf 配置文件

```ini
[mysqld]
# 基础配置
port = 3306
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
default-time-zone = '+08:00'

# 连接配置
max_connections = 500
max_connect_errors = 1000
wait_timeout = 28800
interactive_timeout = 28800

# 缓冲配置
innodb_buffer_pool_size = 1G
innodb_buffer_pool_instances = 4
innodb_log_buffer_size = 64M

# 日志配置
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow.log
long_query_time = 2
log_error = /var/log/mysql/error.log

# InnoDB 配置
innodb_flush_log_at_trx_commit = 1
innodb_lock_wait_timeout = 50
innodb_flush_method = O_DIRECT

# 安全配置
local_infile = 0
skip-symbolic-links

[client]
default-character-set = utf8mb4

[mysql]
default-character-set = utf8mb4
```

## Redis 配置优化

### redis.conf 配置文件

```conf
# 基础配置
bind 0.0.0.0
port 6379
daemonize yes
pidfile /var/run/redis/redis-server.pid
logfile /var/log/redis/redis-server.log

# 内存配置
maxmemory 512mb
maxmemory-policy allkeys-lru

# 持久化配置
save 900 1
save 300 10
save 60 10000
appendonly yes
appendfsync everysec

# 安全配置
requirepass your_redis_password
rename-command FLUSHALL ""
rename-command FLUSHDB ""
rename-command CONFIG ""

# 性能配置
tcp-backlog 511
tcp-keepalive 300
timeout 0
```

## 数据备份

### MySQL 备份

#### 手动备份

```bash
# 全库备份
mysqldump -u root -p ai_business_planner > backup_$(date +%Y%m%d).sql

# 压缩备份
mysqldump -u root -p ai_business_planner | gzip > backup_$(date +%Y%m%d).sql.gz
```

#### 自动备份脚本

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/data/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
MYSQL_USER="root"
MYSQL_PASS="your_password"
DATABASE="ai_business_planner"

mkdir -p $BACKUP_DIR

mysqldump -u$MYSQL_USER -p$MYSQL_PASS $DATABASE | gzip > $BACKUP_DIR/$DATABASE_$DATE.sql.gz

# 删除 7 天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete
```

#### 添加定时任务

```bash
crontab -e

# 每天凌晨 2 点备份
0 2 * * * /path/to/backup.sh
```

### Redis 备份

```bash
# 手动触发 RDB 快照
redis-cli -a your_password BGSAVE

# 复制 RDB 文件
cp /var/lib/redis/dump.rdb /data/backup/redis/dump_$(date +%Y%m%d).rdb
```

## 数据恢复

### MySQL 恢复

```bash
# 解压并恢复
gunzip < backup_20240101.sql.gz | mysql -u root -p ai_business_planner
```

### Redis 恢复

```bash
# 停止 Redis
sudo systemctl stop redis

# 替换 RDB 文件
cp /data/backup/redis/dump_20240101.rdb /var/lib/redis/dump.rdb

# 启动 Redis
sudo systemctl start redis
```

## 常见问题

### 1. MySQL 连接数过多

```sql
-- 查看当前连接
SHOW PROCESSLIST;

-- 查看最大连接数
SHOW VARIABLES LIKE 'max_connections';

-- 临时修改
SET GLOBAL max_connections = 1000;
```

### 2. Redis 内存不足

```bash
# 查看内存使用
redis-cli -a your_password INFO memory

# 手动清理过期键
redis-cli -a your_password SCAN 0 COUNT 1000
```

### 3. 字符集问题

确保数据库、表、连接都使用 `utf8mb4`：

```sql
-- 查看数据库字符集
SHOW CREATE DATABASE ai_business_planner;

-- 修改表字符集
ALTER TABLE messages CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```
