# 后端部署文档

## 技术栈

- NestJS 11.0+
- Prisma 5.22+
- MySQL 8.0+
- Redis 7.0+
- TypeScript 5.9+

## 环境要求

- Node.js >= 18.0.0
- npm >= 9.0.0 或 pnpm >= 8.0.0
- MySQL >= 8.0
- Redis >= 6.0

## 本地开发

### 1. 安装依赖

```bash
cd backend
npm install
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env` 并修改配置：

```env
# 应用配置
PORT=3000
NODE_ENV=development

# 数据库配置 (MySQL)
DATABASE_URL=mysql://用户名:密码@localhost:3306/ai_business_planner

# Redis 配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# JWT 配置
JWT_SECRET=your-super-secret-jwt-key-change-in-production
JWT_EXPIRES_IN=7d

# 硅基流动 AI 配置
SILICONFLOW_API_KEY=your-api-key
SILICONFLOW_BASE_URL=https://api.siliconflow.cn/v1
SILICONFLOW_MODEL=Qwen/Qwen2.5-7B-Instruct

# 配额配置
QUOTA_FREE_DAILY_LIMIT=1
QUOTA_PRO_DAILY_LIMIT=10
```

### 3. 初始化数据库

```bash
# 生成 Prisma Client
npx prisma generate

# 运行数据库迁移
npx prisma migrate dev

# 填充种子数据
npx prisma db seed
```

### 4. 启动开发服务器

```bash
npm run start:dev
```

访问 http://localhost:3000/api

## 生产部署

### 方式一：PM2 部署（推荐）

#### 1. 安装 PM2

```bash
npm install -g pm2
```

#### 2. 构建项目

```bash
npm run build
```

#### 3. 创建 PM2 配置文件

创建 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [{
    name: 'business-planner-backend',
    script: 'dist/main.js',
    instances: 'max',
    exec_mode: 'cluster',
    autorestart: true,
    watch: false,
    max_memory_restart: '1G',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    }
  }]
}
```

#### 4. 启动服务

```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

### 方式二：Docker 部署

#### 1. 创建 Dockerfile

```dockerfile
FROM node:18-alpine AS builder

WORKDIR /app

COPY package*.json ./
COPY prisma ./prisma/

RUN npm ci

COPY . .
RUN npx prisma generate
RUN npm run build

FROM node:18-alpine

WORKDIR /app

COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/prisma ./prisma

ENV NODE_ENV=production
ENV PORT=3000

EXPOSE 3000

CMD ["node", "dist/main.js"]
```

#### 2. 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  backend:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=mysql://root:password@mysql:3306/ai_business_planner
      - REDIS_HOST=redis
      - REDIS_PORT=6379
      - JWT_SECRET=your-jwt-secret
      - SILICONFLOW_API_KEY=your-api-key
    depends_on:
      - mysql
      - redis
    restart: always

  mysql:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=password
      - MYSQL_DATABASE=ai_business_planner
    volumes:
      - mysql_data:/var/lib/mysql
    restart: always

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data
    restart: always

volumes:
  mysql_data:
  redis_data:
```

#### 3. 构建并运行

```bash
docker-compose up -d --build
```

### 方式三：Kubernetes 部署

#### 1. 创建 ConfigMap

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: business-planner-config
data:
  NODE_ENV: "production"
  PORT: "3000"
  DATABASE_URL: "mysql://root:password@mysql-service:3306/ai_business_planner"
  REDIS_HOST: "redis-service"
  REDIS_PORT: "6379"
```

#### 2. 创建 Secret

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: business-planner-secret
type: Opaque
stringData:
  JWT_SECRET: "your-jwt-secret"
  SILICONFLOW_API_KEY: "your-api-key"
```

#### 3. 创建 Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: business-planner-backend
spec:
  replicas: 3
  selector:
    matchLabels:
      app: business-planner-backend
  template:
    metadata:
      labels:
        app: business-planner-backend
    spec:
      containers:
      - name: backend
        image: business-planner-backend:latest
        ports:
        - containerPort: 3000
        envFrom:
        - configMapRef:
            name: business-planner-config
        - secretRef:
            name: business-planner-secret
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
```

## 数据库迁移

### 生产环境迁移

```bash
# 生成迁移文件
npx prisma migrate dev --name your-migration-name

# 生产环境执行迁移
npx prisma migrate deploy
```

## 环境变量说明

| 变量名 | 说明 | 必填 |
|--------|------|------|
| PORT | 服务端口 | 是 |
| NODE_ENV | 运行环境 | 是 |
| DATABASE_URL | MySQL 连接字符串 | 是 |
| REDIS_HOST | Redis 主机地址 | 是 |
| REDIS_PORT | Redis 端口 | 是 |
| REDIS_PASSWORD | Redis 密码 | 否 |
| REDIS_DB | Redis 数据库索引 | 否 |
| JWT_SECRET | JWT 密钥 | 是 |
| JWT_EXPIRES_IN | JWT 过期时间 | 是 |
| SILICONFLOW_API_KEY | 硅基流动 API Key | 是 |
| SILICONFLOW_BASE_URL | 硅基流动 API 地址 | 是 |
| SILICONFLOW_MODEL | 使用的模型 | 是 |
| QUOTA_FREE_DAILY_LIMIT | 免费用户每日配额 | 否 |
| QUOTA_PRO_DAILY_LIMIT | 专业用户每日配额 | 否 |

## 常见问题

### 1. 数据库连接失败

检查 MySQL 是否运行，连接字符串是否正确：
```bash
mysql -u root -p -h localhost -P 3306
```

### 2. Redis 连接失败

检查 Redis 是否运行：
```bash
redis-cli ping
```

### 3. Prisma Client 未生成

重新生成：
```bash
npx prisma generate
```

### 4. 内存不足

调整 PM2 内存限制或增加服务器内存。
