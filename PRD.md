# **商业策划机 - 产品功能需求文档（PRD）**

## **1\. 产品定义**

### **1.1 产品名称**

商业策划机

### **1.2 产品定位**

AI驱动的商业策划顾问工具，帮助创业者将零散想法转化为系统化商业方案。

### **1.3 核心价值主张**

- **消除信息差**：让用户了解行业最先进做法
- **AI落地指导**：明确互联网和AI如何应用到具体项目
- **强制结果交付**：避免无限对话，确保输出可执行文档

### **1.4 目标用户**

- 有创业想法但缺乏系统规划能力的个人
- 需要商业计划书对接资源的中小企业主
- 探索新业务线的企业决策者

## **2\. 用户角色与权限**

**表格**

| 角色  | 权限范围 |
| --- | --- |
| **游客** | 仅可浏览案例展示，查看预设案例的完整内容 |
| **注册用户（免费版）** | 创建项目、进行对话、生成文档、查看历史 |
| **注册用户（专业版）** | 更高项目配额、更长对话轮次、高级功能 |

## **3\. 核心功能模块**

### **3.1 案例展示系统**

**功能描述**向游客和注册用户展示标准化的商业策划案例，建立产品价值认知。

**具体内容**

- 案例卡片展示：行业类型、项目简介、核心亮点
- 案例详情页：完整的4份文档展示（商业设计、功能设计、评价表、风险提示）
- 案例分类：餐饮、零售、SaaS、电商、教育等垂直领域

**业务规则**

- 游客点击案例可直接查看完整内容，无需登录
- 案例内容定期更新，展示最新行业趋势
- 案例仅作参考，不可编辑或复制创建

### **3.2 项目管理系统**

**功能描述**用户可同时管理多个独立的商业项目，每个项目拥有独立的生命周期和数据空间。

**核心功能**

**表格**

| 功能  | 说明  |
| --- | --- |
| 创建项目 | 输入项目名称，选择行业类型，进入首轮对话 |
| 项目列表 | 网格/列表视图，显示项目名称、状态、最后更新时间 |
| 项目搜索 | 按项目名称关键词筛选 |
| 项目删除 | 删除项目及所有关联数据，需二次确认 |
| 项目切换 | 点击项目卡片进入对应项目的对话界面 |

**项目状态定义**

**表格**

| 状态  | 说明  | 可执行操作 |
| --- | --- | --- |
| 对话分析中 | 正在进行AI对话分析，未生成最终文档 | 继续对话、查看历史、删除项目 |
| 已生成版本 | 已完成文档生成，项目锁定 | 查看文档、重新生成新版本、删除项目 |

**业务规则**

- 项目之间数据完全隔离，对话历史、生成文档互不干扰
- 每个项目独立计算对话轮次
- 已完成的项目（已生成版本）不可继续对话，仅可查看或重新生成
- 重新生成将创建新版本，保留历史版本记录

### **3.3 AI对话系统**

**功能描述**用户与AI商业顾问进行多轮对话，逐步梳理商业逻辑，最终生成标准化文档。

**对话流程**

**plain**

复制

第1轮：方向确认（强制）

↓

第2-99轮：深度分析（商业、市场、产品、运营）

↓

第100轮或用户主动触发：生成文档

**首轮强制确认机制**

**表格**

| 环节  | 要求  |
| --- | --- |
| 用户输入 | 描述创业想法（可长可短） |
| AI响应 | 复述理解、提出确认问题、明确方向 |
| 用户确认 | 确认或修正方向 |
| 未确认处理 | 若方向不明确，AI继续引导，但系统记录风险 |

**业务规则**

- 每项目最多100轮对话，达到上限强制进入文档生成
- 免费用户每日最多创建1个项目（即每日1次完整策划机会）
- AI回复需采用引导式风格，避免一次性输出过多信息
- 每10轮提醒剩余轮次，第80轮提示准备收尾，第95轮强制准备生成

### **3.4 文档生成系统**

**功能描述**基于对话内容，自动生成4份标准化商业文档。

**输出文档清单**

**表格**

| 文档  | 内容要求 | 用途  |
| --- | --- | --- |
| **商业设计说明书** | 商业模式、市场分析、运营策略、发展阶段 | 对外融资、团队共识 |
| **互联网软件功能设计说明** | 产品架构、功能清单、AI应用场景、数据流 | 技术开发依据 |
| **项目评价表** | SWOT分析、可行性评分、关键指标、行业对比 | 自我评估、决策参考 |
| **项目风险提示表** | 风险清单、等级评估、应对策略、预警指标 | 风险管控 |

**文档版本管理**

**表格**

| 功能  | 说明  |
| --- | --- |
| 版本生成 | 每次点击"生成"创建新版本，保留历史 |
| 版本对比 | 查看不同版本间的差异 |
| 版本回退 | 恢复到历史版本（仅文本，不恢复对话状态） |
| 版本标签 | 支持为重要版本添加自定义标签 |

**业务规则**

- 仅输出Markdown格式，不提供PDF、Word等其他格式
- 文档内容可编辑（注册用户），编辑后保存为新版本
- 生成文档后项目锁定，不可继续对话

### **3.5 文档查看与编辑系统**

**功能描述**提供文档的阅读、复制、编辑功能。

**查看模式**

**表格**

| 模式  | 说明  |
| --- | --- |
| 渲染视图 | Markdown渲染后的美观排版，适合阅读 |
| 原始视图 | 原始Markdown文本，适合复制编辑 |

**编辑功能**

- 仅支持已生成版本状态的文档
- 仅注册用户可编辑
- 编辑后保存为新版本，不覆盖原版本
- 支持全屏编辑模式

**导出功能**

- 复制Markdown到剪贴板
- 下载.md文件

### **3.6 用户账户系统**

**功能描述**管理用户身份、配额和账户信息。

**注册/登录**

- 支持邮箱注册、手机注册、第三方登录（微信/钉钉）
- 游客模式可浏览案例，创建项目必须登录

**账户信息**

- 用户名、头像、绑定邮箱/手机
- 注册时间、最后登录时间

**配额管理（免费版）**

**表格**

| 配额项 | 限制  | 重置周期 |
| --- | --- | --- |
| 每日项目数 | 1个  | 每日0点 |
| 每项目对话轮次 | 100轮 | 不重置，项目级 |
| 文档生成次数 | 不限  | \-  |

**配额提示**

- 创建项目时检查当日剩余配额
- 配额不足时提示升级专业版或明日再来
- 对话过程中实时显示已用轮次/剩余轮次

## **4\. 页面功能详述**

### **4.1 项目列表页（首页）**

**布局结构**

- 左侧边栏：用户头像、新建项目按钮
- 顶部：搜索框
- 主区域：项目卡片网格 + 案例展示区域

**项目卡片元素**

- 项目名称（用户自定义）
- 状态标签（对话分析中/已生成版本）
- 最后更新时间
- 行业类型图标
- 删除按钮（悬停显示）

**空状态**

- 新用户：显示欢迎语 + 引导创建首个项目 + 推荐查看案例
- 无搜索结果：提示未找到项目，建议修改关键词

**案例展示区域**

- 标题：看看别人的商业策划
- 案例卡片：行业标签、简短描述、查看按钮
- 点击案例卡片进入案例详情页

### **4.2 对话页**

**布局结构**

- 左侧边栏：返回按钮、当前项目信息、状态标签、文档快捷入口
- 中部：对话区域（历史消息 + 输入框）
- 右侧：文档预览区

**对话区域**

- 消息气泡：用户消息（右）、AI消息（左）
- AI消息支持：打字机效果逐字显示、暂停/继续、重新生成
- 输入框：多行文本、发送按钮、字数提示
- 快捷操作：常用问题快捷按钮（如"分析竞争对手"、"设计盈利模式"）

**状态差异显示**

**表格**

| 状态  | 对话区 | 文档区 | 底部操作 |
| --- | --- | --- | --- |
| 对话分析中 | 可输入、显示历史 | 占位提示"完成对话后查看" | 发送按钮 |
| 已生成版本 | 只读历史、不可输入 | 4份文档可切换查看 | 重新生成按钮 |

**轮次提示**

- 对话区顶部显示：当前轮次 / 100轮
- 接近上限时变色提醒（80轮黄色、95轮红色）

### **4.3 文档全屏页**

**入口**

- 对话页右侧文档区点击"查看完整文档"
- 项目列表页已生成版本项目的文档快捷入口

**页面元素**

- 顶部：文档标题、返回按钮、版本选择器、复制按钮、编辑按钮
- 中部：文档内容（渲染视图/原始视图切换）
- 底部：版本信息、编辑时间

**视图切换**

- 渲染视图：美观排版、表格渲染、代码高亮
- 原始视图：纯Markdown文本、适合复制

**版本选择器**

- 下拉列表显示所有版本（v1、v2...）
- 当前版本高亮
- 选择历史版本后显示"回退到此版本"按钮

### **4.4 设置页**

**账户信息区**

- 头像、用户名、邮箱、手机
- 修改密码、绑定/解绑第三方账号

**配额展示区**

- 今日已用项目数 / 每日上限
- 项目总数统计
- 当前套餐类型（免费版/专业版）

**升级入口**

- 专业版权益对比
- 购买/升级按钮

## **5\. 业务流程规则**

### **5.1 项目生命周期**

**plain**

复制

创建项目 → 首轮方向确认 → 深度对话分析 → 生成文档 → 项目锁定

↑ |

└────────── 重新生成新版本（保留历史）───────────┘

**状态转换规则**

- 创建后默认进入"对话分析中"
- 用户主动点击"生成文档"或达到100轮 → 进入"已生成版本"
- "已生成版本"状态下点击"重新生成" → 创建新版本，状态保持
- 删除项目 → 彻底移除所有数据

### **5.2 对话强制规则**

**表格**

| 规则  | 说明  |
| --- | --- |
| 首轮确认 | 第1轮必须完成方向确认，否则标记为高风险项目 |
| 轮次上限 | 100轮强制结束对话，进入文档生成 |
| 频率限制 | 单用户每分钟最多20轮对话 |
| 配额限制 | 免费用户每日1个项目 |

### **5.3 文档生成规则**

**表格**

| 触发条件 | 处理方式 |
| --- | --- |
| 用户主动点击"生成文档" | 立即生成，基于当前对话内容 |
| 达到100轮对话 | 自动触发生成，提示用户"已达上限，正在生成" |
| 生成中 | 显示进度提示，完成后通知用户 |
| 生成失败 | 提示错误原因，提供重试按钮 |

## **6\. 内容质量要求**

### **6.1 AI对话质量**

**表格**

| 维度  | 要求  |
| --- | --- |
| 引导性 | 每轮回复应包含1-3个引导性问题，推动用户思考 |
| 结构化 | 复杂内容使用分点、表格、框架呈现 |
| 专业性 | 使用标准商业术语（商业模式画布、TAM/SAM/SOM等） |
| 实用性 | 建议需具体可执行，避免空泛概念 |
| 及时性 | 结合最新行业趋势和AI技术进展 |

### **6.2 文档质量标准**

**商业设计说明书**

- 市场数据需标注来源或说明为估算
- 发展阶段需明确里程碑和关键指标

**互联网软件功能设计说明**

- MVP方式，尽量简化
- 避免冗余功能非核心功能

**项目评价表**

- SWOT等类型的分析需具体，避免泛泛而谈
- 评分需说明评分依据
- 行业对比需指明对标企业

**项目风险提示表**

- 风险识别需覆盖市场、技术、运营、法律、财务5个维度
- 每个风险需有明确的应对策略
- 预警指标需可量化监测

## **7\. 异常处理**

### **7.1 用户侧异常**

**表格**

| 场景  | 处理方式 |
| --- | --- |
| 首轮方向始终不明确 | AI最多引导3轮，仍不明确则标记并继续，但降低生成质量预期 |
| 用户频繁切换项目 | 正常处理，但提示"建议专注完成一个项目" |
| 用户重复提问 | AI识别并提示"该问题已讨论，建议关注XX方面" |
| 用户输入敏感内容 | 过滤并提示"请聚焦商业讨论" |

### **7.2 系统侧异常**

**表格**

| 场景  | 处理方式 |
| --- | --- |
| AI响应超时 | 提示"响应较慢，请稍候"或"重新生成" |
| 生成文档失败 | 提示错误，保留对话记录，允许重试 |
| 网络中断 | 本地缓存输入内容，恢复后自动重发 |

## **8\. 运营支持功能**

### **8.1 案例管理系统（后台）**

- 案例创建、编辑、上下架
- 案例分类管理
- 案例浏览数据统计

### **8.2 用户管理系统（后台）**

- 用户列表、详情查看
- 配额调整（运营补偿）
- 项目数据查看（脱敏）

### **8.3 内容审核机制**

- 用户输入关键词过滤
- 生成文档质量抽检
- 用户反馈收集（文档质量评分）
